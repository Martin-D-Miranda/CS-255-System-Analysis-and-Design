# CS-255-System-Analysis-and-Design
Project One and Two
DriverPass Project ReflectionProject
Summary: The DriverPass project was designed for a driving school client that needed a more efficient, centralized system to manage student bookings, training packages, and administrative tasks. I designed the system to streamline the scheduling process while ensuring that roles—such as the Owner and IT Officer—maintained secure administrative control.
Strengths: I believe I did particularly well in creating logical UML diagrams that clearly communicated complex administrative workflows to the client.  
Future Revisions: If I could revise my work, I would add more detail to the error-handling processes in my activity diagrams to show how the system recovers from unexpected inputs.
User Needs: I interpreted user needs by focusing on the primary pain points, such as the manual management of appointments, and implemented those as automated functional requirements. It is vital to consider user needs during design because a system that fails to solve the user's core problem will not be adopted, regardless of how technically sophisticated it is.
Software Design Strategy: Moving forward, my approach to software design will prioritize a modular, Agile strategy, focusing on constant communication with the client and iterative improvements based on feedback.
